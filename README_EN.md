## What is Self-System?

Self-System is not just another AI chatbot. It is an **AI-driven self-growing system**:

You describe what you need in natural language → AI generates code in real time → auto-deploys to a Docker container → your system grows new capabilities.

Every conversation makes your digital alter ego more powerful — it can adopt your tools, your habits, your workflows, and ultimately become your extension in the digital world.

## Key Features

![conversation-feature](docs/image/1772506820038-github-conversation-feature.png)

- **Conversation = Feature** — Describe what you need, AI auto-generates frontend pages, backend APIs, and routes, deployed instantly
- **Total Freedom, Zero Architecture Constraints** — Pages, APIs, cron jobs, scrapers, data pipelines… add whatever you want
- **Skill System** — Define reusable skills in Markdown, abstracting API integrations and automations into AI-readable documents
- **Docker-in-Docker Architecture** — Control container manages the App container, with hot code reloading and isolated yet customizable environments
- **Real-time Feedback** — WebSocket streams AI execution progress and status in real time
- **Auto-repair** — Automatically triggers repair workflows when errors are detected
- **PWA Support** — Installable on desktop/mobile home screen with offline caching

## How It Differs

![kingcrab-vs-lobster](docs/image/1772509767745-github-kingcrab-vs-lobster.png)

| Dimension | Self-System | Traditional Agent Platforms |
|-----------|-------------|----------------------------|
| Core Paradigm | AI generates code, system self-grows | Message routing / workflow orchestration |
| Customization Depth | Unlimited (generates arbitrary code) | Within Plugin API or preset component boundaries |
| Extension Method | Conversation = Feature | SDK / drag-and-drop orchestration |
| Target Users | Programmers | Everyone |

> If traditional Agent platforms are the Android of personal AI, then Self-System is the AI Linux for programmers. You don't download apps — you compile the entire operating system. Except the compiler is natural language.

               │ Docker API

## Skill System

![skill-system](docs/image/1772506820038-github-skill-system.png)

Skills in Self-System are **Markdown files**, not code modules:

```markdown
---
name: wechat-automation
description: Auto-send WeChat messages
---
## Webhook URL: https://xxx
## Format: { "userName": "group_name", "message": "content" }
```

When the executor is AI, the optimal instruction format shifts from "code" to "documentation".

## Quick Start

### Prerequisites

- Docker & Docker Compose
- A Supabase project (for data persistence)
- AI Model API Key (multiple models supported, see configuration below)

### 1. Clone and Configure

```bash
git clone https://github.com/MaskerPRC/self-system.git
cd self-system
cp .env.example .env
```

Edit the `.env` file:

```env
# Supabase
SUPABASE_URL=your_supabase_url
SUPABASE_ANON_KEY=your_supabase_anon_key

# AI Model (pick one, priority: ZHIPU > MINIMAX > QWEN > Proxy)
# Option 1: Zhipu GLM5
ZHIPU_API_KEY=your_key
# Option 2: MiniMax
MINIMAX_API_KEY=your_key
# Option 3: Qwen
QWEN_API_KEY=your_key
# Option 4: Proxy mode
CLAUDE_CODE_URL=https://your-proxy-url
CLAUDE_CODE_KEY=your_key

# Authentication
AUTH_USERNAME=yourname
AUTH_PASSWORD=yourpassword
```

### 2. Initialize Database

Run `supabase/schema.sql` in your Supabase project to create the required tables.

### 3. Docker Deployment (Recommended)

```bash
docker compose up -d --build
```

After startup:

| Service | Container Port | Host Port | URL |
|---------|---------------|-----------|-----|
| Control Panel (API + Web + WebSocket) | 3000 | 42617 | http://localhost:42617 |
| App Frontend | 5174 | 53781 | http://localhost:53781 |
| App API | 3001 | 53783 | http://localhost:53783 |

### 4. Local Development

```bash
# Install dependencies
pnpm install
cd control/server && pnpm install && cd ../..
cd control/frontend && pnpm install && cd ../..
cd app/server && pnpm install && cd ../..
cd app/frontend && pnpm install && cd ../..

# Start all services
npm run dev
```

## Workflow

```
User input → Save message → Queue dispatch → Build prompt (with skills)
    → AI generates code → Determine output type:
        ├── [RESPONSE]   → Direct reply
        ├── [PAGE_INFO]  → Write files → Restart app container → Verify → Auto-repair
        ├── [SKILL_INFO] → Save skill config
        └── [FILE_INFO]  → Generate file
    → WebSocket pushes results in real time
```

## API Reference

### Conversations

```
GET    /api/conversations           # List conversations
POST   /api/conversations           # Create conversation
PATCH  /api/conversations/:id       # Update conversation title
DELETE /api/conversations/:id       # Delete conversation
```

### Messages

```
GET    /api/conversations/:id/messages   # Get messages
POST   /api/conversations/:id/messages   # Send message (triggers AI generation)
```

### Pages

```
GET    /api/pages                   # List all pages
GET    /api/pages/active            # Get active pages
DELETE /api/pages/:id               # Delete page
```

### Skills

```
GET    /api/skills                  # List skills
GET    /api/skills/:id              # Get skill details
POST   /api/skills                  # Create skill
PATCH  /api/skills/:id              # Update skill
DELETE /api/skills/:id              # Delete skill
```

### System

```
GET    /api/health                  # Health check
POST   /api/heartbeat               # Page heartbeat report
GET    /api/queue/status            # Queue status
```

## Philosophy

- **Code is consumable** — Don't debug, regenerate
- **Skills are knowledge abstractions** — Tell AI "what to do", not "how to do it"
- **The system is a living organism** — It evolves with every conversation
- **Full autonomy** — Your alter ego, your rules
